# Dhithra v2 — AI Research Document System

Simple single-repo setup. **One Python file** runs everything. React frontend is compiled into `static/` and served by FastAPI.

```
dhithra_v2/
├── main.py              ← ENTIRE backend (auth + docs + blocks + agents + serve SPA)
├── requirements.txt     ← Python deps (minimal)
├── render.yaml          ← One-click Render deploy
├── build.sh             ← Build script (Render runs this)
├── .env.example         ← Copy to .env
├── static/              ← Generated by `npm run build` (gitignored)
└── frontend/
    ├── package.json
    ├── vite.config.js
    ├── index.html
    └── src/
        └── main.jsx     ← ENTIRE React app (single file)
```

## Deploy to Render (free)

### Step 1 — Get a free OpenRouter API key
1. Go to **https://openrouter.ai/keys**
2. Sign up → Create key
3. Free models include `mistralai/mistral-7b-instruct:free`, `google/gemma-3-12b-it:free`, and more — see **https://openrouter.ai/models?q=free**

### Step 2 — Push to GitHub
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/YOUR_USERNAME/dhithra.git
git push -u origin main
```

### Step 3 — Deploy on Render
1. Go to **https://render.com** → New → Web Service
2. Connect your GitHub repo
3. Render auto-detects `render.yaml` — click **Apply**
4. Go to **Environment** tab → Add:
   - `OPENROUTER_API_KEY` = your key from Step 1
5. Click **Deploy** — done!

Your app will be live at `https://dhithra.onrender.com` (or similar).

> **Note on free tier**: Render's free tier spins down after 15 minutes of inactivity. First request after spin-down takes ~30 seconds. SQLite data persists within a deployment but resets on redeploy. For persistent data, add a **Render Disk** ($1/month) or use their **PostgreSQL** free tier and set `DATABASE_URL`.

## Run locally

### Backend
```bash
cd dhithra_v2

# Create virtual environment
python -m venv venv
source venv/bin/activate   # Windows: venv\Scripts\activate

# Install deps
pip install -r requirements.txt

# Configure
cp .env.example .env
# Edit .env — add OPENROUTER_API_KEY

# Run
uvicorn main:app --reload --port 8000
# API: http://localhost:8000/api
# Docs: http://localhost:8000/docs
```

### Frontend (dev mode)
```bash
cd frontend
npm install
npm run dev
# Opens: http://localhost:5173
# Proxies /api to localhost:8000
```

### Build frontend (production-like)
```bash
cd frontend
npm run build
# Outputs to ../static/
# Now http://localhost:8000 serves the full app
```

## API Reference

```
POST /api/auth/login             Login or register (Name + Email)
GET  /api/auth/me                Current user

GET  /api/documents              List documents
POST /api/documents              Create document
GET  /api/documents/{id}         Get document with blocks
PATCH /api/documents/{id}        Update title/status
DELETE /api/documents/{id}       Delete document

POST /api/documents/{id}/blocks  Add block
PATCH /api/blocks/{id}           Update block
DELETE /api/blocks/{id}          Delete block

POST /api/agents/run             Run an agent
GET  /api/agents/list            List agents

GET  /api/analytics/summary      Request stats
GET  /api/health                 Health check
```

### Agent request
```json
{
  "agent": "generate | think | structure | reason | research | cite",
  "prompt": "your prompt",
  "document_id": "uuid (optional)",
  "block_id": "uuid (for reason agent)"
}
```

## Adding Persistent Storage on Render

### Option A — Render Disk ($1/month)
1. Render dashboard → your service → **Disks** → Add Disk
2. Mount path: `/data`
3. Set `DATABASE_URL=sqlite:////data/dhithra.db` in Environment

### Option B — Render PostgreSQL (free tier)
1. Render dashboard → New → PostgreSQL → Free plan
2. Copy the **Internal Database URL**
3. Set it as `DATABASE_URL` in your web service environment
4. Add `psycopg2-binary` to `requirements.txt`
